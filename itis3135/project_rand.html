Project Overview

This Web Application is for a client outside of the class and the primary goal of the application is to promote their professional business. As of right now, the business does not have a copy of their current menu online yet. They hope that through this project, they can have a website that promotes their business, showcase their food, provide contact information, provide online ordering websites, and have a current menu for customers to read. This website is mainly for customer use.

That being said, the goal of this project is to have 6 pages,

  • A HomePage
  • Food Gallery Page
  • Online Ordering Services Page
  • Food Menu Page
  • Gluten and Pregnancy Options
  • Contact Us Page

Page Design

HomePage:

  1. The title will be “Sushi Star NC Project Website” with a subheading saying “Homepage”.
  2. This page will be the index page for the project. It will have a mini slideshow showcasing some of the food options. It will also have a mini summary of the purpose of the website and why it is a “project”.
  3. Customers will be the users of this page.
  4. A small slideshow showcasing the food products and a mini summary of the purpose of the slideshow will be part of the homepage.
  5. There would be no need for the user to input data in this page.
  6. There would be no need to validate anything.
  7. There will be no button on this page other than the buttons to check the html and the css.
  8. The html and css buttons will direct the user to another page that certifies that it is working.
  9. The plan is to use jquery for the slideshow.

Food Gallery Page:

  1. The title will be “Sushi Star NC Project Website” with a subheading saying “Food Gallery Page”.
  2. This page will be one of the first features in this project. After clicking this link, the user will be moved to another page that showcases Sushi Star’s food products.
  3. Customers will be using this page.
  4. There will be a summary at the top of the page then 2 sections below the summary, each being a slideshow showcasing the food products.
  5. There would be no need to put user data in this page.
  6. There would be no need to validate anything.
  7. There will be no buttons other than the buttons to check the html code and the css.
  8. No new features other than the 2 slideshows.
  9. The plan is to use jquery for the two slideshows and moving text.

Online Ordering Services Page:

  1. The title will be “Sushi Star NC Project Website” with a subheading saying “Online Ordering Services Page”.
  2. The goal of the page is to give information to customers regarding their current delivery services. There will be images of the delivery service logo and attached to the image should be a link to the delivery page.
  3. The goal of this page is for customers.
  4. This page will not be asking for any data.
  5. Since the page will not be asking for data, there will be no need to validate something that will not exist.
  6. There will be buttons to take the user to the delivery service page as well as hyperlinks to the instructions on how to use a delivery service.
  7. The buttons will take the user to the delivery service page.
  8. No new features will be added to the page other than the buttons.
  9. The plan is to use just html and css because we won’t need to use js.

Food Menu Page:

  1. The title will be “Sushi Star NC Project Website” with a subheading saying “Food Menu Page”.
  2. The goal of this page is to have an image of the current menu as well as a link to the pdf version of the menu
  3. This page is for customers.
  4. This page will not be taking in any data.
  5. There will be no need to validate anything in this page
  6. Other than the CSS and the Html validation buttons, The only plan is to have buttons that will take the user to a pdf version of the menu.
  7. The buttons will take the user to a pdf version of the menus
  8. No new features other than the buttons will be implemented.
  9. No need to use js, just the default css will work.

Gluten and Pregnancy Options Page:

  1. The title will be “Sushi Star NC Project Website” with a subheading saying “Gluten and Pregnancy Options”.
  2. The goal of this page is to have a list of pregnancy options and gluten-free options. I would also like to add a feature where the user will be able to pick their food preferences, and click a button that generates a random food option that fits their preferences.
  3. This page is for customers.
  4. This page will be taking data through checkboxes
  5. This page will not need validation since it will just have checkboxes
  6. Other than the CSS and the Html validation buttons, The only plan is to have buttons that will alert the user to a sushi roll based on their preferences.
  7. The buttons will use a js file to alert the user their choice.
  8. No new features other than the buttons implemented.
  9. There will be a separate JS file that will hold the function to print out the alerts for the web page.

Contact Us Page:

  1. The title will be “Sushi Star NC Project Website” with a subheading saying “Contact Us”.
  2. The goal of this page is to have the business’s contact information and also have a form that will allow user to reach out to the business.
  3. This page is for customers or anyone who would like to reach out to the business.
  4. This page will be taking text data and save it so the business can read their message and reach out to them accordingly.
  5. This page will need to validate the email addresses and require names.
  6. Other than the CSS and the Html validation buttons, The only plan is to have a form like feature on the page that will allow the user to submit their reason of contact as well as their information so the business can reach them.
  7. There will be a js file to take all the information but there will be more research in this process.
  8. No new features other than the buttons implemented.
  9. There will be a separate JS file that will hold the function to take all of the information inputted in the form.
